## 워크플로우

Blog 채널은 리서처·라이터·이미지 메이커·어셈블러 4개의 서브 에이전트로 구성됩니다. URL 또는 주제 키워드를 입력하면 팩트 수집부터 SEO 최적화 초안 작성, 이미지 프롬프트 생성, 최종 완성본 조립까지 순서대로 자동 실행됩니다.

```
researcher → writer → image-maker → assembler → reviewer
```

| 단계            | 설명                                                 |
| --------------- | ---------------------------------------------------- |
| **researcher**  | 팩트·출처 수집 → `research/research.md`              |
| **writer**      | SEO 블로그 초안 작성 (persona·seo·style 가이드 적용) |
| **image-maker** | 커버+섹션별 이미지 프롬프트 생성                     |
| **assembler**   | 이미지 마커·CTA 삽입 → 완성본                        |
| **reviewer**    | 톤·팩트·플랫폼 스펙 최종 검수                        |

## 기본 체인 (매번 자동 실행)

```
core/planner
  → dermatology-specialist  (specialist: dermatology 일 때)
  → blog-researcher
  → blog-writer
```

출력: `deliverables/blog.md` (상태: `<!-- 상태: 초안 -->`)

## 비주얼/완성 체인 (별도 요청 시)

```
blog-image-maker → blog-assembler
```

출력: `deliverables/blog.md` in-place 업데이트 (상태: `<!-- 상태: 이미지+CTA 삽입 완료 -->`)

## 검수 체인 (별도 요청 시)

```
core/reviewer
```

출력: `reviews/review-report.md`

## 에이전트 역할

| 에이전트           | 역할                                       | 체인   |
| ------------------ | ------------------------------------------ | ------ |
| `blog-researcher`  | 주제별 팩트·출처 수집                      | 기본   |
| `blog-writer`      | 팩트 기반 SEO 블로그 초안 (2,000-3,500자)  | 기본   |
| `blog-image-maker` | 커버+본문 이미지 프롬프트 생성 + 자체 검수 | 비주얼 |
| `blog-assembler`   | 이미지 마커·CTA 삽입 → blog.md in-place    | 비주얼 |
| `core/reviewer`    | 톤·팩트·플랫폼 스펙 검수                   | 검수   |

## 공통 입력 파일

모든 blog 에이전트는 실행 전 `ACTIVE_OUTPUT_DIR`를 먼저 확인합니다.

- `tenants/.active`
- `tenants/{tenant-slug}/brand.md`
- `tenants/{tenant-slug}/compliance.md`
- `tenants/{tenant-slug}/keywords.md`
- `{ACTIVE_OUTPUT_DIR}/briefs/master.md`
- `{ACTIVE_OUTPUT_DIR}/briefs/compliance.md` (있을 때)
- `{ACTIVE_OUTPUT_DIR}/briefs/channel/blog.md` (있을 때)
- `{ACTIVE_OUTPUT_DIR}/research/trends.md` (있을 때)
- `{ACTIVE_OUTPUT_DIR}/research/competitors.md` (있을 때)

## 단계별 출력

| 단계 | 에이전트                          | 출력 파일                       |
| ---- | --------------------------------- | ------------------------------- |
| 0    | `core/planner`                    | `briefs/master.md`              |
| 0.5  | `dermatology-specialist` (조건부) | `briefs/compliance.md`          |
| 1    | `blog-researcher`                 | `research/research.md`          |
| 2    | `blog-writer`                     | `deliverables/blog.md` (초안)   |
| 3    | `blog-image-maker`                | `visuals/blog-image-prompts.md` |
| 4    | `blog-assembler`                  | `deliverables/blog.md` (완성본) |
| 5    | `core/reviewer`                   | `reviews/review-report.md`      |

## 가이드 참조

`blog-writer` 실행 시 아래 가이드를 반드시 읽고 적용합니다.

| 가이드        | 역할                                                                     | 파일 경로                             |
| ------------- | ------------------------------------------------------------------------ | ------------------------------------- |
| persona-guide | 요청 목적에 따라 4가지 페르소나(후기형/교육형/비교형/에세이형) 선택 기준 | `agents/blog/guides/persona-guide.md` |
| seo-guide     | 네이버/Google SEO 최적화 기준 (제목, 키워드 밀도, 메타 설명)             | `agents/blog/guides/seo-guide.md`     |
| style-guide   | 공통 문체 가이드 — 톤, 금지 표현, 문장 길이 등                           | `agents/blog/guides/style-guide.md`   |

## 핸드오프 규칙

- `core/planner`가 반드시 먼저 실행되어 `briefs/master.md`를 생성해야 합니다.
- `blog-researcher`는 planner 출력이 있을 때만 시작할 수 있습니다.
- `blog-writer`는 `research/research.md` 외 사실을 임의로 작성하면 안 됩니다.
- `blog-assembler`는 이미지 마커·CTA 삽입과 체크리스트 수정 외에는 writer 초안을 그대로 보존합니다.
- `core/reviewer`는 `blog.md` 완성본이 존재한 뒤 실행합니다.
- `briefs/master.md`에 `specialist: dermatology`가 있으면 `dermatology-specialist`가 researcher/writer보다 먼저 실행되어야 합니다.
