Instagram 채널은 리서처·라이터·이미지 메이커·어셈블러 4개의 서브 에이전트로 구성됩니다. 주제 키워드를 입력하면 팩트 수집·슬라이드 구조 제안부터 카드뉴스 카피 작성, 이미지 프롬프트 생성·검수, 최종 완성본 조립까지 순서대로 자동 실행됩니다.

## 워크플로우

```
researcher → writer → image-maker → assembler → reviewer
```

| 단계 | 설명 |
| --- | --- |
| **researcher** | 팩트 시트 + 슬라이드 구조 제안 + 해시태그 풀 수집 |
| **writer** | 슬라이드 카피 + 영문 이미지 프롬프트 + 캡션 + 해시태그 작성 |
| **image-maker** | writer 프롬프트 검수·보정 (샘플-퍼스트 원칙 적용) |
| **assembler** | CTA 확정 + 캡션 품질 체크 → 완성본 |
| **reviewer** | 톤·팩트·플랫폼 스펙 최종 검수 |

## 기본 체인 (매번 자동 실행)

```
core/planner
  → dermatology-specialist  (specialist: dermatology 일 때)
  → instagram-researcher
  → instagram-writer
```

출력:
- `deliverables/instagram.md` (슬라이드 카피 + 영문 이미지 프롬프트, 상태: `<!-- 상태: 초안 -->`)
- `visuals/instagram-image-prompts.md` (이미지 프롬프트 별도 저장)

## 비주얼/완성 체인 (별도 요청 시)

```
instagram-image-maker → instagram-assembler
```

출력: `visuals/instagram-image-prompts.md` 검수·보정 + `deliverables/instagram.md` in-place 업데이트 (상태: `<!-- 상태: 완성 -->`)

### 샘플-퍼스트 원칙

`instagram-image-maker` 최초 실행 시 전체 슬라이드를 한번에 만들지 않고 커버 슬라이드(1장) 프롬프트 + 방향 설명을 먼저 제안합니다. 사용자가 방향을 승인한 뒤 전체 슬라이드로 진행합니다.

- 샘플 저장 경로: `visuals/sample-preview-prompt.md`
- 샘플 단계에서 Figma MCP 호출 금지
- "바로 다 만들어줘" 요청 시 샘플 단계 생략 가능

## 검수 체인 (별도 요청 시)

```
core/reviewer
```

출력: `reviews/review-report.md`

## 에이전트 역할

| 에이전트 | 역할 | 체인 |
| --- | --- | --- |
| `instagram-researcher` | 팩트 시트 + 슬라이드 구조 제안 + 해시태그 풀 (3개+ 소스) | 기본 |
| `instagram-writer` | 슬라이드 카피 + 영문 이미지 프롬프트 + 캡션 + 해시태그 | 기본 |
| `instagram-image-maker` | writer 프롬프트 검수·보정 + visuals/ 파일 확정 | 비주얼 |
| `instagram-assembler` | CTA brand.md 연결 + 캡션 품질 체크 → 완성본 | 비주얼 |
| `core/reviewer` | 톤·팩트·플랫폼 스펙 검수 | 검수 |

## 공통 입력 파일

모든 instagram 에이전트는 실행 전 `ACTIVE_OUTPUT_DIR`를 먼저 확인합니다.

- `tenants/.active`
- `tenants/{tenant-slug}/brand.md`
- `tenants/{tenant-slug}/compliance.md`
- `tenants/{tenant-slug}/keywords.md`
- `{ACTIVE_OUTPUT_DIR}/briefs/master.md`
- `{ACTIVE_OUTPUT_DIR}/briefs/compliance.md` (있을 때)
- `{ACTIVE_OUTPUT_DIR}/briefs/channel/instagram.md` (있을 때)
- `{ACTIVE_OUTPUT_DIR}/research/trends.md` (있을 때)
- `{ACTIVE_OUTPUT_DIR}/research/competitors.md` (있을 때)

## 단계별 출력

| 단계 | 에이전트 | 출력 파일 |
| --- | --- | --- |
| 0 | `core/planner` | `briefs/master.md` |
| 0.5 | `dermatology-specialist` (조건부) | `briefs/compliance.md` |
| 1 | `instagram-researcher` | `research/research.md` |
| 2 | `instagram-writer` | `deliverables/instagram.md` (초안) + `visuals/instagram-image-prompts.md` |
| 3 | `instagram-image-maker` | `visuals/instagram-image-prompts.md` (검수·보정) |
| 4 | `instagram-assembler` | `deliverables/instagram.md` (완성본) |
| 5 | `core/reviewer` | `reviews/review-report.md` |

## 가이드 참조

`instagram-writer` 실행 시 아래 가이드를 반드시 읽고 적용합니다.

| 가이드 | 역할 | 파일 경로 |
| --- | --- | --- |
| persona-guide | 요청 목적에 따라 4가지 페르소나 선택 기준 | `agents/instagram/guides/persona-guide.md` |
| platform-guide | 인스타그램 알고리즘·해시태그 전략 | `agents/instagram/guides/platform-guide.md` |
| style-guide | 공통 카피 가이드 — 톤, 금지 표현, 슬라이드 문장 길이 | `agents/instagram/guides/style-guide.md` |

## 핸드오프 규칙

- `core/planner`가 반드시 먼저 실행되어 `briefs/master.md`를 생성해야 합니다.
- `instagram-researcher`는 슬라이드 구조 제안 테이블을 `research.md`에 포함해야 하며, `instagram-writer`는 이를 슬라이드 레이아웃 출발점으로 사용합니다.
- `instagram-writer`는 `research/research.md` 외 사실을 임의로 작성하면 안 됩니다.
- `instagram-writer`는 슬라이드 카피와 영문 이미지 프롬프트를 동시에 작성합니다. `[IMAGE: ...]` 마커 방식 금지 — 프롬프트를 인라인으로 직접 작성하고 `visuals/instagram-image-prompts.md`에도 저장합니다.
- `instagram-image-maker`는 writer 프롬프트를 검수·보정하는 역할이며, 처음부터 새로 생성하지 않습니다.
- `instagram-assembler`는 CTA 확정과 캡션 품질 체크만 담당하며, 슬라이드 카피와 이미지 프롬프트는 건드리지 않습니다.
- `briefs/master.md`에 `specialist: dermatology`가 있으면 `dermatology-specialist`가 researcher/writer보다 먼저 실행되어야 합니다.
