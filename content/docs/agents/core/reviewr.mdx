모든 채널 콘텐츠 생성이 완료된 뒤 실행합니다. 브랜드 톤 일관성, 팩트 정확성, 플랫폼별 스펙 준수 여부를 체크하고 `reviews/review-report.md`에 검수 결과를 저장합니다.

## 트리거

콘텐츠 생성 완료 후 **반드시** 실행합니다.

## 검수 항목

### 1. 브랜드 톤 일관성

- "요"체 사용 확인
- 친근하고 실용적인 톤 유지
- AI틱 표현 검출 및 수정 요청

  | 금지 표현                        | 대체 방향                                 |
  | -------------------------------- | ----------------------------------------- |
  | "혁신적인", "획기적인", "놀라운" | 구체적 수치로 대체 ("작업 시간 50% 단축") |
  | "완벽한", "최고의", "게임체인저" | 사실 기반 표현으로 대체                   |

### 2. 팩트 체크

`briefs/master.md`의 원본 콘텐츠 요약과 대조하여 인물명·숫자·인용구·기술 용어 정확성을 검증합니다.

### 3. 플랫폼별 스펙 검증

| 채널        | 주요 기준                                           |
| ----------- | --------------------------------------------------- |
| 블로그      | 3,000-5,000자, H2/H3 구조, SEO 키워드 포함          |
| 인스타그램  | 슬라이드 5-7장, 캡션 2,200자 이내, 해시태그 20-30개 |
| 뉴스레터    | 15,000-20,000자, Q&A 인터뷰 구조                    |
| X 스레드    | 트윗 10개, 각 280자 이내                            |
| 링크드인    | 1,500자 이내, 전문가 톤                             |
| 유튜브 대본 | 타임스탬프 포함, 시각 자료 표시                     |
| 쇼츠 대본   | 3개, 각 60초 (150-180자), 훅→본문→CTA 구조          |

### 4. 의료광고 클레임 검증 (피부과 콘텐츠)

- 금지 표현 미사용 확인 ("완치", "100%", "반드시")
- 효과 수치에 출처 표기 확인
- 안전 문구 포함 여부 ("개인별 결과는 다를 수 있어요")
- 환자 경험담/전후 사진 언급 없음

### 5. 언어 자연스러움

| language | 적용 기준                                          |
| -------- | -------------------------------------------------- |
| `ko`     | 번역체 패턴 검출, 맞춤법·띄어쓰기                  |
| `en`     | 한국어 직역체 감지, 관사·전치사, AI 패턴 금지 표현 |

### 6. Figma 디자인 자산 검수

- `designs/design-report.md` 존재 및 Figma URL 확인
- 레이어 구조 편집 가능 여부 (단일 이미지 업로드 금지)
- 썸네일/카드뉴스/CTA 슬라이드 존재 여부

## 출력 파일

`{ACTIVE_OUTPUT_DIR}/reviews/review-report.md`

```
검수 일시
전체 요약 (검수 대상 수 / 승인 / 수정 필요)
채널별 상세 결과 (상태, 분량, 톤, 팩트, 수정 사항)
언어 검수 결과 표
팩트 체크 결과 표
최종 결론 (배포 가능 / 재검수 필요 / 재작성 권장)
```
